AutoEncoders
Create an encoding which captures all the crucial details of the input

Use this encoding to re-construct the image

Compare reconstructed image with the original

Adjust Encoder to generate better encoding, and Decoder to decode better

Repeat.
Add Data + GPU Compute Power as per Taste
It's super easy to implement. No, really!
Training
We use One Encoder and Two Decoders for Training

For all images of Person A:
    Warp -> Encode -> Decode with Decoder A -> Diff(Original, Output ) -> Update Network

For all images of Person B:
    Warp -> Encode -> Decode with Decoder B -> Diff(Original, Output ) -> Update Network
Image from KDNuggets
Because of the setup, the decoder is reconstucting the original images as truly as possible
including expressions, lighting, accessories etc.
One "Really"  Clever Trick
After we are happy enough with the performance of the encoder and
decoder (quantified by the Loss Function), we are ready with our model!

Give the encoder an image of person A, but use Decoder B to reconstruct image

The Encoder has been trained to create an encoding for both Person A as well
as Person B so encoder behaves normally

Decoder B has been trained to reconstruct Person B from an encoding, so assumes
encoding generated is simply another encoding of a Warped Person B image.

Thus, we trick our AutoEncoder to perform a facial swap.
