How memory works!

Every process has it's own pagetable, which lists it's memory mapping

Since each process has it's own PT, the memory allocated to one process is not
visible to another

To access any memory location, a "privilege check" is done.





Authorised memory access attempted == All is well!
Unauthorised memory access attempted == Throw exception and fail

How Fast is Fast?
Modern CPUs are tremendously fast!

The bottleneck for a CPU is waiting 
for data to process.

It's literally faster for a CPU to calculate 
extra information and discard whatever 
is not needed.

So that's exactly what they do.

Meltdown
Intel CPUs have speculative execution that can access non-process memory WITHOUT having 
a privilege check.
if v:
   w = kern_mem[address] # If we get here,fault
   x = w&0x100           # Mask everything except first bit
          y = user_mem[x]                               #   This will bring 0x100 or 0x000 into cache
Now, we can read user_mem[0x000] and user_mem[0x100].

One of them is cached so we will be much faster.


Congratulations, we've just read a bit of memory!
The world has melted!
Am I Affected?
Almost every Intel manufactured chip 
since ~1995 is/was affected by Meltdown. 

Includes Cloud providers like AWS and GCE as well
Mitigation
Kernel Page Table Isolation (KPTI) a.k.a
Kernel Address Isolation to have Side-channels Efficiently Removed (KAISER)
Simply put, make the kernel page table completely invisible to userspace. So no priviledged
memory is accessible to userspace processes.

Performance Hits: ~5% to 25% depending on how CPU bound your processes are

Spectre

Instrument : A probing array which we specifically tell the CPU not to cache 


We attempt to access Instrument[A[x]] where x is a different process memory block

Suppose A[x] == 4, so Instrument[4] is pulled into cache.

Now, we access I[0], I[1], I[2]... 

I[4] is in cache, so access time is MUCH faster than for I[0], I[1] etc.

So we can deduce that A[x], which should not be accessible to me holds value 4.
(This is called playing the instrument!)
Am I Affected?
Spectre is much harder to exploit, but affects nearly
EVERY CPU out in the wild today.

In fact, it's called Spectre because "As it is not easy to fix, 
it will haunt us for quite some time."
Mitigation
Retpoline:
Return trampolines allow us to not speculate when we make 
indirect jumps. This allows the specific attack mentioned before
to be mitigated. However, since Spectre is a whole class of attacks,
it's difficult to say for sure that Retpolines are a fix-it-all. 

Basically, replace indirect jumps with infinite loops to avoid speculation.
This has minimal performance impact as well! \m/

Retpolines were developed at Google for x86 and being adapted for 
other architectures.


There are other approaches being taken for mitigating Spectre, but all of
them target a subset of attack vectors opened up by Spectre. Spectre is MUCH
harder to attack to be useful (and extremely slow as of now), but opens up a
fundamental flaw in how CPUs work.
